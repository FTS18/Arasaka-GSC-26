# JANRAKSHAK GSC-26 OPERATIONS MANUAL

This manual provides standard operating procedures for the Janrakshak Tactical Console.

---

## 1. System Overview
The Janrakshak GSC-26 is a centralized command console for humanitarian logistics. The platform is designed to minimize response latency by automating the triage and dispatch process.

## 2. Operational Hierarchy

### Level 1: Administrator (Full Command)
- Full system configuration.
- Audit log oversight.
- Manual intervention in dispatch logic and volunteer assignment.
- Disaster Mode activation.
- Verification of reported incidents.

### Level 2: Volunteer (Field Response)
- Registering of specialized skills (Medical, Logistics, etc.).
- Accepting assigned missions.
- Submitting proof-of-completion (Photos/Telemetry).
- Real-time status updates via the mobile console.

### Level 3: User (Public Access)
- Reporting incidents and needs for community support.
- Monitoring request status.
- Contributing resources to warehouses (Donor functionality).

---

## 3. Standard Procedures

### Incident Reporting and Triage
1. **Intake**: Reports arrive via the Public Portal (raw text or images) from Users.
2. **Extraction**: The Integrated Intelligence Engine extracts location, urgency, and category.
3. **Prioritization**: The system assigns a Priority Score (0-100) based on vulnerability and time-decay.
4. **Validation**: An Administrator reviews the AI-extracted data for accuracy before dispatch.

### Mission Assignment
- **Auto-Assignment**: In high-urgency scenarios, the Administrator uses the "Auto-Assign" feature to immediately dispatch the best-matched volunteer.
- **AI Recommendation**: Review the AI-generated rationale before manual assignment to understand geospatial and skill-based matching logic.

---

## 4. Disaster Protocols (Disaster Mode)
Disaster Mode should only be activated by an Administrator during large-scale emergencies.

**Effects of Activation**:
- The UI transitions to a High-Contrast Emergency Theme.
- Automated priority boosts are applied to all life-saving categories.
- Re-routing of volunteers from low-urgency tasks to critical hotspots.

---

## 5. Ethics and Transparency
- Every action is recorded in the immutable Audit Trail.
- Personal identifiable information (PII) must be handled according to humanitarian privacy standards.
- AI scores are transparent and can be queried for rationale at any time.
